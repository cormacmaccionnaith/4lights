# The Four Lights — project notes

## Deployment & branches (IMPORTANT)
- **`main` is the trunk and the live/production branch.**
- **Always push to `main`.** Do not develop on or push to a separate feature
  branch unless the user explicitly asks for one.
- When the user says **"push live"** (or "push to main"), it means commit and
  push to **`main`**.
- Railway serves the site; the production service deploys from `main`.

## Build
- The site is generated by a zero-dependency Node script:
  `node src/build.js` renders `src/content/*.js` through `src/templates/*.js`
  into static HTML at the repo root. **Commit the generated HTML.**
- Rebuild after any content or template change, then push to `main`.

## Editing content
- All copy lives in `src/content/`:
  - `swims.js` — the four swims (story, crossing, vitals, distance, map).
  - `site.js` — series name, homepage sections, rules & safety, contact.
- Route-map images live in `assets/img/maps/<slug>.<ext>`; the build uses a
  chart image when present and falls back to a stylised SVG map otherwise.
